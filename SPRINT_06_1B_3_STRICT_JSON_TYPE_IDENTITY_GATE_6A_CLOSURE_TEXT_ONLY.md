# Sprint 06.1B.3 — Strict JSON Type Identity and Gate 6A Closure

## PM decision

Sprint 06.1B.2 produced a valid economic evidence set, but the current checker still accepts type-invalid JSON values because ordinary Python equality treats booleans, integers and numerically equal floats as equal.

Observed accepted rehashed mutations:

```text

manifest.parameter_selection_authorized_bool: false -> 0
```

All changed non-manifest members had valid recomputed SHA-256 values. The failure is semantic type coercion, not hashing.

This is the final Gate 6A closure. Do not implement OHLC replay in this commit.

## Task 1 — Reject JSON floats and non-standard numeric constants

Update the strict JSON boundary in:

```text
src/bybit_grid/backtest/neutral_grid/evidence.py
src/bybit_grid/backtest/neutral_grid/serialization.py
```

Requirements:

1. State-machine evidence JSON must not contain JSON floating-point tokens at all.
2. `1.0`, `3.3e1`, `-0.0` and similar tokens must fail.
3. JSON `NaN`, `Infinity` and `-Infinity` must fail.
4. Duplicate-key rejection remains.
5. Plain JSON integers, booleans, strings, arrays, objects and null remain supported.
6. Decimal accounting values remain canonical strings, as currently implemented.

Recommended parser behavior:

```python
json.loads(
    text,
    object_pairs_hook=...,
    parse_float=reject_json_float,
    parse_constant=reject_nonfinite_constant,
)
```

Use stable `EvidenceError` codes, for example:

```text
json_float_forbidden
nonfinite_json_number_forbidden
```

In `canonical_json_bytes()`:

- use `allow_nan=False`;
- fail on Python `float` values rather than silently serializing them;
- preserve current canonical sorting, separators, UTF-8 and final newline.

Do not change the canonical representation of existing valid v2 evidence.

## Task 3 — Replace type-loose artifact equality with exact canonical byte identity

The checker currently parses artifacts and later uses ordinary dict/list equality. Replace the evidence acceptance boundary with exact expected canonical bytes.

After strict parsing and canonical-input validation, independently build the frozen expected records:

```python
expected_inputs, expected_results, expected_ledger, expected_cycles = build_evidence_records()
```

Require:

```text
scenario_inputs.jsonl bytes == jsonl_bytes(expected_inputs)
scenario_results.jsonl bytes == jsonl_bytes(expected_results)
ledger_events.jsonl bytes == jsonl_bytes(expected_ledger)
completed_cycles.jsonl bytes == jsonl_bytes(expected_cycles)
```

Do not rely on:

```python
actual_rows == expected_rows
```

because that is type-loose for bool/int and int/float.

For JSON documents, construct the exact expected object and require exact canonical bytes:

```text
state_machine_run_status.json
state_machine_contract_audit.json
scenario_catalog.json
scenario_audit.json
reproducibility_audit.json
```

Examples:

```python
artifact_bytes == canonical_json_bytes(expected_object)
```

The independent scenario replay and `audit_simulation_result()` checks must remain. Exact bytes supplement them; they do not replace accounting audits.

Reports remain exact byte contracts:

```text
synthetic_scenario_report.md
risk_budget_readiness_report.md
```
