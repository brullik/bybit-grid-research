## Task 3 — Enforce an exact manifest contract

Update the checker and shared evidence constants.

The manifest must contain exactly this key set and no other keys:

```text

```

Requirements:

```text
missing key -> fail
extra key -> fail
wrong type -> fail
wrong value -> fail
manifest self-hash -> fail
sha256 keys != exact 11 non-manifest members -> fail
```

The manifest is self-excluded, so this exact-key contract is mandatory.

Require the manifest bytes themselves to equal canonical JSON serialization of the parsed, validated manifest:

```python
manifest_bytes == canonical_json_bytes(parsed_manifest)
```

Thus whitespace, duplicate-key representations or non-canonical key ordering cannot masquerade as canonical persisted evidence.

Duplicate JSON keys must fail. The standard `json.loads()` last-value behavior is not sufficient. Add a strict JSON loader using `object_pairs_hook` or equivalent and reject duplicate keys in all JSON artifacts and JSONL rows.

## Task 4 — Enforce canonical persisted JSON and JSONL bytes

After semantic validation, require exact canonical bytes.

### JSON documents


### JSONL documents

For:

```text
scenario_inputs.jsonl
scenario_results.jsonl
ledger_events.jsonl
completed_cycles.jsonl
```

require:

```python
stored_bytes == jsonl_bytes(validated_expected_rows)
```

Reject, even after member hash is recomputed:

- added blank lines;
- pretty-print whitespace;
- changed key order;
- missing final newline;
- duplicate JSON keys;
- non-canonical Decimal strings introduced through a changed payload;
- any extra row or field.

Semantic equality alone is not enough for this frozen deterministic evidence pack.
