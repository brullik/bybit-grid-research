## Observed defects to close

### A. Environment-dependent generated-file test

The current test scans the entire working directory:

```python
Path(".").glob("**/*.zip")
Path(".").glob("**/*.jsonl")
```

It fails when the owner has valid historical review packs or when the current owner-side run generates JSONL files. Do not require the owner to delete old packs to pass tests.

### B. CLI mismatch

The owner workflow uses:

```text
make_state_machine_review_pack.py --output <zip>
check_state_machine_review_pack.py --zip <zip> --run-id <id>
```

The implementation currently uses `--pack-path` and a positional checker path. Support the owner form and retain backward compatibility.

### C. Semantic checker gap

After recomputing hashes, the current checker accepts altered manifest, contract audit, reproducibility audit, results and ledger evidence. Hash validation alone is not an independent audit.

### D. Replay/reproducibility booleans are not fully evidence-derived

The runner does not compare persisted results to a fresh replay when producing `all_scenarios_replay_match_bool`, and reproducibility booleans are written as constants.
