## Why two OHLC paths are required

A 1-minute candle contains open, high, low and close but does not contain the temporal order in which high and low occurred.

Implement two deterministic **minimal-turn path hypotheses**:

```text
OPEN_HIGH_LOW_CLOSE
OPEN_LOW_HIGH_CLOSE
```

These hypotheses visit each reported extreme once. They do **not** reconstruct the real tick path and do **not** bound arbitrary additional oscillations inside the candle.

Never label the resulting min/max as the true best case, true worst case or complete intrabar bound. Use the exact name:

```text
minimal_path_ambiguity_envelope
```

## Existing entry-time contract

Preserve the project’s existing no-lookahead entry rule:

```text
entry_time_ms = ((signal_time_ms // 60000) + 1) * 60000
```

The signal candle is never replayed. The first replay candle must start at `entry_time_ms`.

## Task 1 — Strict candle model

Implement frozen dataclass/enums with runtime validation.

### `CandleSource`

At minimum:

```text
synthetic_1m
bybit_trade_kline_1m
```

Enum instances only; strings must not be silently accepted.

### `OhlcCandle1m`

Fields:

```text
category: str
symbol: str
open_time_ms: int
open: Decimal
high: Decimal
low: Decimal
close: Decimal
closed_bool: bool
source: CandleSource
```

Validation:

```text
category is exactly str "linear"
symbol is a non-empty stripped str
open_time_ms is int, not bool
open_time_ms >= 0
open_time_ms % 60000 == 0
closed_bool is exactly bool and must be true
source is CandleSource
OHLC values are finite positive Decimal, never float/int/bool
low <= open <= high
low <= close <= high
low <= high
if low == high, open == high == low == close
```

Properties:

```text
close_boundary_ms = open_time_ms + 60000
```

Do not use wall-clock time in the model. Closed-candle status is explicit and deterministic.

### `FundingObservation`

Fields:

```text
time_ms: int
funding_rate: Decimal
mark_price: Decimal
```

Validation:

```text
time_ms is int, not bool, >= 0 and minute-aligned
funding_rate is finite Decimal
mark_price is finite positive Decimal
```

### `MinimalPathPolicy`

Exactly:

```text
open_high_low_close
open_low_high_close
```

## Task 2 — Candle sequence contract

Add validation for replay inputs:

```text
at least one candle
all candles same category and symbol
strictly increasing open_time_ms
no duplicate timestamps
exact 60000 ms contiguity
first candle open_time_ms == entry_time_ms
all candles closed
```

Do not interpolate missing candles and do not silently sort or deduplicate. Fail closed.

A difference between previous close and next open is an observable gap, not a data error. The next candle’s open event must process that gap through the state machine.

## Task 3 — Canonical minimal path generation

For each candle generate one of these price tuples:

```text
OHLC policy: open -> high -> low -> close
OLHC policy: open -> low -> high -> close
```

Rules:

- Remove only consecutive duplicate prices.
- Do not remove a later repeated price separated by another point.
- Do not round prices.
- Do not snap to grid levels or base price.
- Preserve exact Decimal values.
- For every generated `PriceEvent` belonging to a candle, use that candle’s `open_time_ms` as the time bucket.
- `sequence_id` provides the total intrabar order.
- This timestamp is a candle-bucket timestamp, not a claim about the actual time high/low occurred.

Add helpers returning both the normalized price tuple and whether the two minimal policies are distinct for that candle.

## Task 4 — Funding ordering contract

Funding is optional in 06.2A but must be deterministic when supplied.

Rules:

```text
funding observations sorted strictly by time
no duplicate funding time
funding time must match a replay candle open boundary
funding time must be > entry_time_ms
funding time must be < final candle close_boundary_ms
funding at entry_time_ms is rejected, not silently charged
funding at a candle boundary is processed before that candle’s price-path events
funding uses its explicit mark_price
```

This matches the existing project rule that funding at the exact entry boundary is excluded.

Do not infer mark price from trade OHLC in this sprint.

## Task 5 — Fixed-policy replay

Implement a replay function that accepts:

```text
NeutralGridConfig
entry_time_ms
validated tuple/list of OhlcCandle1m
one MinimalPathPolicy for every candle, or one fixed policy for the sequence
optional FundingObservation sequence
```

Behavior:

1. Instantiate a fresh `NeutralGridReferenceEngine`.
2. Generate globally increasing sequence IDs beginning at 1.
3. At each candle boundary, process a matching funding observation first.
4. Process the selected minimal price path.
5. If the engine terminates, stop immediately; do not submit later candle/funding events.
6. Return a detached frozen replay result.
7. Run `audit_simulation_result()` on the final engine result and fail if it does not pass.

Do not duplicate accounting formulas in the OHLC layer.
