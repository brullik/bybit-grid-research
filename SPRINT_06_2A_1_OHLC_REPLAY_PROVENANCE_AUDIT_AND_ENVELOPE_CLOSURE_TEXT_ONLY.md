## Task 1 — Strict adapter provenance

At the OHLC adapter boundary require:

```text
config is exactly a NeutralGridConfig instance
config.category == candle category == "linear"
config.symbol == every candle symbol
config.symbol is a stripped non-empty str
candle symbol is a stripped non-empty str
```

Update `OhlcCandle1m.__post_init__` so:

```text
symbol == symbol.strip()
```

is required, not merely `symbol.strip() != ""`.

Do not alter `NeutralGridConfig` or other frozen Gate 6A behavior unless strictly necessary. Adapter-level validation is sufficient.

`validate_candle_sequence()` must type-check every object before dereferencing its fields and must fail with a stable `ValueError`, not an incidental `AttributeError`.

## Task 2 — Strict enumeration-cap contract

`max_exact_ambiguous_candles` must be:

```text
int
not bool
>= 0
```

Reject floats, strings, booleans and negative values with stable `ValueError` messages before any enumeration or candle processing.

Retain `MinimalPathEnumerationCapExceededError` only for a valid cap that is smaller than the actual ambiguous-candle count.

At minimum retain detached tuples:

The result's:

```text

entry_time_ms

```

must be derivable exactly from this retained evidence.

## Task 4 — Strict generated-event model

Replace free-form generated event semantics with an exact runtime contract.

Add an enum such as:

```text
ReplayEventKind.price
ReplayEventKind.funding
```

`GeneratedReplayEvent` must validate:

```text
sequence_id: int, not bool, >= 1
time_ms: int, not bool, >= 0, minute-aligned
kind: ReplayEventKind instance
candle_index: int, not bool, >= 0
price/mark_price: finite positive Decimal
funding_rate: finite Decimal for funding events, None for price events
```

For a price event:

```text
funding_rate is None
```

For a funding event:

```text
funding_rate equals the source FundingObservation.funding_rate
price field equals the explicit source mark_price
```

Do not accept strings such as `"price"` as substitutes for the enum.

## Task 5 — Exact expected event-stream reconstruction

Implement a deterministic pure helper that reconstructs the candidate event schedule from:

```text
source_candles
path_policies
source_funding_observations
```

It must enforce:

```text
funding before price at a shared candle boundary
canonical normalized path nodes
strict sequence IDs starting at 1
correct candle_index
exact Decimal prices and funding rates
```

The actual replay event tuple is a prefix of this schedule only when termination occurs. The accepted prefix must end at the exact price event that caused termination. No event with a later sequence ID may be present.

## Task 6 — Independent replay audit

Refactor the implementation so the public audit can perform a fresh replay without recursive audit calls.

An acceptable architecture is:

```text
_execute_ohlc_replay_core(...) -> OhlcReplayResult
replay_ohlc_minimal_path(...) -> core execution + required state-machine audit
audit_ohlc_replay_result(...) -> validate retained input + fresh core replay + exact comparison
```

The public audit must independently:

1. Validate source candle sequence and funding observations.
2. Validate config/category/symbol provenance.
3. Recompute path-policy count and ambiguous-candle count.
4. Reconstruct the expected generated event stream.
5. Fresh-replay the retained source evidence through a new engine.
6. Compare the stored replay result to the fresh replay using exact type-aware semantics.
7. Run `audit_simulation_result()` again on the stored nested state-machine result; do not trust only `state_machine_audit_passed_bool`.
8. Check exact category, symbol, entry time, counts, final mark, final PnL, termination flag/reason and generated events.
9. Check funding-before-price ordering at every shared boundary.
10. Check there is no generated event after the termination-causing sequence.
11. Check processed/unprocessed candle counts are derived from generated event evidence.
12. Fail closed with stable failure codes on every exception.

A changed proof flag in the nested state-machine result must be rejected by the rerun state-machine audit.

Do not compare only Python values where `True == 1` can pass. Use exact types/enum identities/Decimal identities.

## Task 7 — Envelope aggregate correctness

Fix construction of `MinimalPathAmbiguityEnvelope`.

Compute independently across all assignment results:

```text
completed_cycle_count_min = min(cycle counts)
completed_cycle_count_max = max(cycle counts)
trading_fees_min_usdt = min(fees)
trading_fees_max_usdt = max(fees)
termination_reasons_observed = exact deterministic sorted set
```

Do not derive cycle min/max from the min-PnL and max-PnL assignments.

Keep PnL min/max tie-break behavior deterministic:

```text
primary key: PnL
secondary key: full policy assignment key, OHLC before OLHC
```

Define and document `path_sensitive_bool` as a recomputed property. It must be true when any material replay outcome differs across assignments, not merely when the cached flag says so. At minimum include:

```text
final_total_pnl_usdt
termination_reason
completed cycle count
cumulative trading fees
signed final position
```
