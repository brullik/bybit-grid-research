# Sprint 06.3A.1 — Contract-Type-Aware Instrument Parsing and Public Smoke Closure

## PM decision

The Sprint 06.3A public smoke reached Bybit successfully. DNS, HTTP 200, server time, single BTCUSDT instrument parsing, and BTCUSDT funding history all passed.

The full linear universe failed with:

```text
PublicBatchError: funding_interval_minutes_out_of_range
```

The current model applies `funding_interval_minutes > 0` to every row before replay eligibility is evaluated. This is too narrow because Bybit `category=linear` contains more than the project's USDT perpetual replay universe.

This sprint is a narrow public-input contract correction. Do not alter accepted state-machine or OHLC replay economics.

## Task 1 — Separate generic instrument validity from replay eligibility

Update:

```text
src/bybit_grid/data/public_batch/models.py
```

### Accepted linear contract types

The strict model must recognize exactly:

```text
LinearPerpetual
LinearFutures
```

Reject unknown contract types without coercion.

### Funding interval source contract

For generic `BybitInstrumentMeta`:

```text
funding_interval_minutes must be exact int
bool is rejected
funding_interval_minutes >= 0
```

Do not convert zero to 480 or any other default.

Do not discard zero. Preserve it as source evidence.

### Replay eligibility

`eligible_for_replay()` must require all of:

```text
contract_type == LinearPerpetual
status == Trading
quote_coin == USDT
settle_coin == USDT
is_pre_listing is False
funding_interval_minutes > 0
```

A `LinearFutures` row with `funding_interval_minutes=0` is valid generic metadata but not replay eligible.

A `LinearPerpetual` row with `funding_interval_minutes=0` may be parsed for diagnosis, but it is not replay eligible and must be surfaced by the universe audit.

Negative values remain invalid.


Definitions:


The audit must fail if:

- symbols are duplicated;
- a row is not an exact `BybitInstrumentMeta`;
- any replay candidate has funding interval zero;
- any replay-eligible row has a non-positive funding interval;
- an unknown contract type is present.

A zero funding interval on `LinearFutures` must be reported but must not by itself fail the replay-universe audit.

Use immutable tuples/mapping proxies where appropriate. Do not return mutable source aliases.
